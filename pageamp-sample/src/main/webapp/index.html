Reactive Logic

  • PageAmp lets you define logic attributes, which don't appear in generated pages, by prefixing them with :.

  • Logic attributes can be used in expressions, surrounded by [[ and ]], anywhere in texts and attribute values.

  • When the value of a logic attribute changes, all dependent expressions are automatically updated.

Isomorphic Pages

  • Expressions work the same in both the client and the server.

  • The server generates pages that are ready for search engine indexing and are quickly displayed to users while the client runtime is loaded.

  • Modern browsers will seamlessly take over logic execution, while all other clients will see an old style website.

Hello Component

Imports

  • PageAmp preprocesses pages in the server. It considers *.htm files as fragments to be included in your pages.

  • The <:import> tag only includes a particular fragment once, while the <:include> tag is replaced by the fragment at each occurrence.

  • Imports are the basis for effective HTML modularization and componentization in PageAmp.

Defines

  • PageAmp's preprocessor lets you define your own parametric custom tags with the <:define> tag.

  • You can use custom tags to avoid repeated blocks of code. Combined with imports, you can also define reusable components.

  • Wrapping existing HTML frameworks as PageAmp kits is quick and convenient. Plus, PageAmp provides its own pagekit.

Hello Data

Data Binding

  • Page elements can be given a data context with the :data attribute. Nested elements inherit the context and can refine it or replace it with their own.

  • If the context is an array, the element is automatically cloned.

  • Logic attributes inside an element can access context data via the data variable. Thanks to reactivity, data changes are automatically reflected in the DOM.

Data Sources

  • PageAmp adds a simple formal representation of dynamic data to plain HTML with the <:datasource> tag.

  • A data source can access existing data and services as well as define a client- or server-side service itself, e.g. for DB persistence.

  • Access to third party services will be automatically relayed through the server in order to securely overcome the same-origin limitations.
